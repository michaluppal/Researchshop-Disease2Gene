# Pipeline Reports

This directory contains generated, report-style, or one-off pipeline investigation artifacts. They are retained for historical traceability and publication/audit context.

These files are **not** the active source of truth for current pipeline architecture or behavior. For current references, use:

- [`../pipeline-contract.md`](../pipeline-contract.md) - canonical pipeline contract.
- [`../internals.md`](../internals.md) - curated maintainer technical notes.
- [`../../audit/AUDIT.md`](../../audit/AUDIT.md) - audit history, accepted tradeoffs, and benchmark notes.

Regenerate report outputs only when intentionally updating the specific investigation they document.
