# ResearchShop Evaluation

This directory contains the evaluation framework for the ResearchShop gene discovery pipeline.

## Directory Structure

```
evaluation/
├── README.md                     # This file
├── ground_truth.json             # Manually curated ground truth (19 MIS-C papers)
├── evaluation_analysis.ipynb     # Main evaluation notebook (recommended)
├── run_evaluation.py             # Command-line evaluation script (alternative)
└── results/                      # Output directory for evaluation results
```

## Ground Truth

The `ground_truth.json` file contains manually curated gene-disease associations for 19 MIS-C research papers.

**Total: 19 papers, 103 gene-paper associations**

## Evaluation Notebook (Recommended)

The `evaluation_analysis.ipynb` Jupyter notebook provides comprehensive evaluation with:

### Features

1. **Ground Truth Validation** - Checks if GT genes actually appear in fetched paper text
2. **Multi-Run Analysis** - 5 runs to measure variance and reproducibility
3. **Ablation Analysis** - Compares validation ON vs OFF from single runs
4. **Error Categorization** - Classifies errors:
   - **False Positives**: hallucination, valid HGNC (GT incomplete), in text but not GT
   - **False Negatives**: in text (missed), not in text (unreachable)
5. **Per-Gene Analysis** - Identifies consistently missed/hallucinated genes
6. **Paper Difficulty Ranking** - Ranks papers by extraction difficulty
7. **Text Source Comparison** - Full-text vs abstract performance
8. **Visualizations** - Charts for all analyses
9. **Thesis-Ready Summary** - Formatted output for Chapter 7

### Environment Setup

First, create the conda environment:

```bash
cd Final_thesis_code/evaluation

# Option 1: Use setup script (recommended)
./setup_env.sh

# Option 2: Manual setup
conda env create -f environment.yml
conda activate researchshop-eval
playwright install chromium
python -m ipykernel install --user --name researchshop-eval --display-name "ResearchShop Eval"
```

### Running the Notebook

1. Set environment variables:
   ```bash
   export GEMINI_API_KEY="your-api-key"
   export ENTREZ_EMAIL="your-email@example.com"
   ```

2. Activate environment and open notebook:
   ```bash
   conda activate researchshop-eval
   jupyter notebook evaluation_analysis.ipynb
   ```

3. Select kernel: **Kernel → Change Kernel → ResearchShop Eval**

4. Run all cells (Kernel → Restart & Run All)

### Output

- `results/evaluation_complete_YYYYMMDD_HHMMSS.json` - Full results
- `results/evaluation_charts.png` - Visualization charts

## Command-Line Script (Alternative)

For quick evaluations without the notebook:

```bash
python run_evaluation.py
python run_evaluation.py --output-dir ./my_results
```

This produces metrics for both validation ON and OFF from a single run.

## What Gets Evaluated

The evaluation runs **only gene discovery** (Stages 1-6), skipping detailed attribute extraction:

1. **Paper Fetching** - Full text via PMC/web scraping, fallback to abstract
2. **Gene Discovery** - Gemini-2.0-Flash extracts gene-variant associations
3. **Validation** - HGNC database + MyGene.info + paper text verification

## Metrics

- **Precision**: TP / (TP + FP) - How many extracted genes are correct
- **Recall**: TP / (TP + FN) - How many ground truth genes were found
- **F1-Score**: Harmonic mean of precision and recall
- **Adjusted Metrics**: Only count GT genes that appear in fetched text (fairer assessment)

## Key Insights

The evaluation distinguishes between:

1. **LLM extraction errors** - Genes in text but not extracted
2. **Data quality issues** - GT genes not present in fetched text
3. **Validation filtering** - Valid genes filtered by heuristics
4. **Hallucinations** - Invalid gene symbols generated by LLM

This separation allows precise measurement of each component's contribution to overall performance.
