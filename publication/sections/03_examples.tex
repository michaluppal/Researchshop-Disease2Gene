\section{Illustrative Examples}
\label{sec:examples}

\subsection{Research Workflow}

ResearchShop is designed for a common literature-curation workflow: a researcher starts from a PubMed query or a curated PMID list, reviews the candidate papers in the desktop interface, selects the papers to process, defines the output columns they need, and receives a spreadsheet of gene--paper associations with provenance and validation metadata. The same backend can also be run from the command line for reproducible scripted analyses.

For example, a researcher studying inflammatory genetics can submit a PMID list and request columns such as \emph{Key Finding}, \emph{Disease Association}, \emph{Statistical Evidence}, and \emph{Conclusion}. The pipeline then fetches open-access full text, parses body text, tables, figures, and supplementary metadata where available, combines PubTator candidates with deterministic HGNC scanning, and asks Gemini to fill the requested schema only for grounded candidate genes. The output includes the extracted fields, citation validation status, NCBI identifiers, candidate source, association type, association group, confidence label, and paper metadata.

\subsection{Audit Artifacts}

Each run writes more than the final CSV. ResearchShop persists a metadata CSV, XLSX workbook, JSON export, candidate audit artifact, drop-debug artifact, and optional pipeline trace. These artifacts are intentionally verbose: they record which genes were proposed, which gates accepted or rejected them, whether citations were grounded in the source text, and whether output rows originated from LLM extraction, PubTator, deterministic seeding, or fallback paths.

This design is important for scientific review. A researcher can inspect why a gene appeared in the output, why another gene was dropped, and whether the supporting sentence was copied from the paper or generated by the LLM. The application therefore treats extraction as an auditable triage step rather than as an opaque classifier.

\subsection{Figure and Table Handling}

Many genomics papers encode primary findings in figures, tables, and supplementary files rather than in prose. ResearchShop parses PMC and Europe PMC JATS XML, extracts structured table text, collects figure captions and image URL candidates, and passes table/caption context into evidence validation. When figure analysis is enabled and images are downloadable, Gemini Vision can inspect figures as an optional signal; when images are unavailable, the pipeline degrades to caption and text evidence rather than failing the run.

The parser path uses ResearchShop-owned fetchers for PubMed, PMC, Europe PMC, and citation services. A small adapter uses \texttt{pubmed\_parser} for selected JATS parsing tasks such as body paragraphs and figure metadata, while ResearchShop keeps its own open-access gates, URL probing, CDN fallback, table extraction, supplementary file handling, PubTator integration, and output schema.

\subsection{Reproducibility Checks}

The repository includes offline tests for the main scientific risk points: XML parsing, figure metadata extraction, citation grounding, gene validation, F2-isoprostane false-positive suppression, association-type grouping, PubTator/NCBI caching, per-paper extraction import compatibility, output writing, and pipeline tracing. These checks do not require a Gemini key or network access and are intended to be run before release builds.

The SoftwareX submission focuses on software architecture, usability, auditability, and reproducibility of the workflow. Historical benchmark scripts and audit records remain in the repository for transparency, but the manuscript does not rely on an expanded external benchmark as a prerequisite for publication.
