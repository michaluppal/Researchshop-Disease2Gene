\section{Illustrative Examples}
\label{sec:examples}

\subsection{Research Workflow}

ResearchShop supports a compact literature-curation loop: users provide a PubMed query, author search, or PMID list; select papers; define schema columns such as \emph{Key Finding}, \emph{Disease Association}, \emph{Statistical Evidence}, and \emph{Conclusion}; and receive spreadsheet-ready gene--paper rows with provenance and validation metadata. For each selected open-access paper, the pipeline parses text, tables, captions, figures, and supplementary metadata where available; combines PubTator, HGNC scanning, and LLM candidate discovery; fills the requested schema only for grounded candidates; and exports citation status, NCBI identifiers, candidate source, association grouping, confidence label, and paper metadata. The same backend can be run through the desktop interface or from the command line for scripted analyses.

Figures~\ref{fig:app_workflow_screens} and~\ref{fig:app_results_screen} show the protocol, query/schema, and results views. Table~\ref{tab:demo_output_trace} uses a local demonstration result file to illustrate the visible data model; it is not an accuracy benchmark.

\subsection{Audit Artifacts}

Each run writes the final CSV plus metadata CSV, XLSX, JSON, candidate-audit, drop-debug, and optional trace files. These artifacts record candidate origin, gate outcomes, citation-grounding status, confidence score, fallback paths, and dropped-row reasons. A reviewer can therefore see why a gene appeared, why another was excluded, and whether a supporting sentence was recovered from the paper rather than generated by the LLM. Extraction failures become reproducible records, not silent omissions.

\subsection{Figure and Table Handling}

Many genomics papers encode primary findings in figures, tables, and supplementary files rather than in prose. ResearchShop parses PMC and Europe PMC JATS XML, extracts structured table text, collects figure captions and image URL candidates, and passes table/caption context into evidence validation. When figure analysis is enabled and images are downloadable, a multimodal Gemini model can inspect figures as an optional signal; when images are unavailable, the pipeline degrades to caption and text evidence rather than failing the run.

The parser path uses ResearchShop-owned fetchers for PubMed, PMC, Europe PMC, and citation services. A small adapter uses \texttt{pubmed\_parser} for selected JATS parsing tasks such as body paragraphs and figure metadata, while ResearchShop keeps its own open-access gates, URL probing, CDN fallback, table extraction, supplementary file handling, PubTator integration, and output schema.
