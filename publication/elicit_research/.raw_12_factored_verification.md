        - paragraph [ref=e109]:
          - text: At Elicit, we want to help researchers derive new insights from existing literature and
          - link "scale up good reasoning" [ref=e110] [cursor=pointer]:
            - /url: https://blog.elicit.com/mission/
          - text: . To achieve this, Elicit uses a range of different large language models to automate the search, extraction and summarization of academic literature. Language models, while extraordinarily powerful, aren't trained to be truth-seeking and are prone to generating false claims about a given source text (i.e hallucinating). At best, this causes our users to waste verifying claims made by Elicit or worse update their thinking based on incorrect information.
        - paragraph [ref=e111]:
          - text: The
          - link "main technique" [ref=e112] [cursor=pointer]:
            - /url: https://arxiv.org/pdf/2203.02155.pdf?ref=blog.elicit.com
          - text: to reduce hallucinations involves using reinforcement learning from human feedback (RLHF) to encourage the model to be more truthful.
          - link "Prior work has" [ref=e113] [cursor=pointer]:
            - /url: https://arxiv.org/abs/2212.09251?ref=blog.elicit.com
          - text: "already established that this can induce sycophancy and power-seeking behavior. We believe that RLHF is not always aligned with the goal of accurate summarization even in current generation models:"
          - link "GPT-4" [ref=e114] [cursor=pointer]:
            - /url: https://cdn.openai.com/papers/gpt-4.pdf?ref=blog.elicit.com
          - text: already surpasses the performance of the average human on many different academic tasks. The model can summarize highly technical information from multiple documents in seconds. A non-expert does not have sufficient domain knowledge to understand the nuances of the summary and correct subtle mistakes. RLHF could in fact induce the model to generate “good looking” summaries that could, for example, present a compelling narrative around a research question which is unsupported by the literature.
        - paragraph [ref=e115]:
          - text: We explore a method we call
          - emphasis [ref=e116]: factored verification
          - text: "to help solve this problem. The technique involves breaking up a summary into smaller pieces (claims, sentences, …) and verifying each one individually. This has 2 advantages:"
        - list [ref=e117]:
          - listitem [ref=e118]:
            - text: •
            - paragraph [ref=e119]: First, verifying a single claim is easier for the model to correct than checking an entire summary.
          - listitem [ref=e120]:
            - text: •
            - paragraph [ref=e121]: Second, this narrower task is also easier for humans to check.
        - paragraph [ref=e122]:
          - text: This technique can be thought of as an application of
          - link "factored cognition" [ref=e123] [cursor=pointer]:
            - /url: https://ought.org/research/factored-cognition?ref=blog.elicit.com
          - text: and inherits the associated safety benefits. This allows the use of weak or weakly aligned models to supervise stronger ones. This also allows the oversight process to scale with model capabilities.
        - paragraph [ref=e124]:
          - text: As part of this method, we construct
          - emphasis [ref=e125]: factored critiques
          - text: by concatenating each false claim and the relevant model reasoning into a single critique. We then ask the model to revise its summary based on this feedback. Importantly, if no errors are found we do not change the summary. This avoids the model “correcting” true statements which is an issue with chain-of-thought-based self-correction. Meta’s concurrent
          - link "chain-of-verification work" [ref=e126] [cursor=pointer]:
            - /url: https://arxiv.org/abs/2309.11495?ref=blog.elicit.com
          - text: can be viewed as the same approach applied to out-of-context hallucination reduction, whereas we focus on in-context hallucination.
        - paragraph [ref=e127]: "Below is an example of a factored critique:"
        - paragraph [ref=e128]: "Our main empirical findings are:"
        - list [ref=e129]:
          - listitem [ref=e130]:
            - text: •
            - paragraph [ref=e131]: Factored verification achieves SOTA the summarization section of HaluEval beating chain of thought prompting.
          - listitem [ref=e132]:
            - text: •
            - paragraph [ref=e133]: When summarizing across multiple academic papers, we find 0.62 hallucinations in the average ChatGPT (16k) summary, 0.84 for GPT-4, and 1.55 for Claude 2.
          - listitem [ref=e134]:
            - text: •
            - paragraph [ref=e135]:
              - text: Asking GPT-4 to generate a critique of its summary and then revise it actually increases hallucinations by 37%. This is consistent with the findings in “
              - link "Large Language Models Cannot Self-Correct Reasoning Yet" [ref=e136] [cursor=pointer]:
                - /url: https://arxiv.org/abs/2310.01798?ref=blog.elicit.com
              - text: ”.
          - listitem [ref=e137]:
            - text: •
            - paragraph [ref=e138]: We ask models to self-correct using factored critiques and find that this lowers the number of hallucinations by an average of 35% to 0.49 for ChatGPT, 0.46 for GPT-4, and 0.95 for Claude 2.
          - listitem [ref=e139]:
            - text: •
            - paragraph [ref=e140]: In the small sample we validated, humans initially missed over half of the true hallucinations which were only found after inspecting the model's reasoning. At the same time, GPT-4 does overestimate the number of hallucinations by about 50%.
          - listitem [ref=e141]:
            - text: •
            - paragraph [ref=e142]: Similar to prior work, we find that generating critiques works better with stronger models (GPT-4) but generating and revising summaries can be done with weaker ones (ChatGPT).
        - paragraph [ref=e143]: This project was done as part of our work on internal tools for testing and improving the accuracy of Elicit. We’re excited about a world where progress in AI safety can at least partially be product-driven. It’s clear that human feedback doesn’t scale and can induce unintended negative effects in frontier models. We hope that our work provides some evidence that scalable AI oversight is becoming increasingly feasible.
